# Feature Specification: 유튜브 동영상 다운로더

**Feature Branch**: `001-video-download`

**Created**: 2026-06-15

**Status**: Draft

**Input**: 프론트엔드에 유튜브 URL 입력 기능이 있음. 입력받은 URL을 이용해서 유튜브 다운로드 기능 실행. 저장옵션 선택(mp4, mp3). 다운로드 진행상황 표시.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - URL 입력 및 동영상 다운로드 (Priority: P1)

사용자가 유튜브 동영상 URL을 입력하면 시스템이 해당 영상의 정보를 조회하고, 사용자가 원하는 형식(mp4 또는 mp3)을 선택한 뒤 다운로드를 시작한다. 다운로드가 완료되면 파일이 지정된 위치에 저장된다.

**Why this priority**: 이것이 서비스의 핵심 가치이자 유일한 기능 흐름이다. 이것이 동작해야 나머지 모든 기능이 의미를 가진다.

**Independent Test**: URL 입력 → mp4 형식 선택 → 다운로드 버튼 클릭 → 파일이 저장소에 생성되는 것을 확인하면 독립적으로 테스트 가능.

**Acceptance Scenarios**:

1. **Given** 사용자가 유효한 유튜브 URL을 입력했을 때, **When** 다운로드 버튼을 클릭하면, **Then** 시스템이 영상 정보(제목, 길이, 썸네일)를 표시하고 다운로드를 시작한다.
2. **Given** 다운로드가 완료되었을 때, **When** 파일 저장 위치를 확인하면, **Then** 선택한 형식(mp4 또는 mp3)으로 파일이 존재한다.
3. **Given** 사용자가 잘못된 URL을 입력했을 때, **When** 다운로드 버튼을 클릭하면, **Then** 명확한 오류 메시지가 표시되고 다운로드는 시작되지 않는다.

---

### User Story 2 - 저장 형식 선택 (Priority: P2)

사용자가 다운로드 전 저장 형식을 선택할 수 있다. 동영상 형식(mp4)과 음원 추출 형식(mp3) 중 하나를 선택한다.

**Why this priority**: 형식 선택은 기본 다운로드 기능이 동작한 이후에 추가 가치를 제공한다. mp4 단일 형식만으로도 MVP가 가능하다.

**Independent Test**: URL 입력 후 mp3 형식을 선택하고 다운로드 완료 시 mp3 파일이 생성되는지 확인.

**Acceptance Scenarios**:

1. **Given** 사용자가 유효한 URL을 입력했을 때, **When** 형식 목록을 확인하면, **Then** mp4와 mp3 옵션이 표시된다.
2. **Given** 사용자가 mp3를 선택하고 다운로드했을 때, **When** 저장 파일을 확인하면, **Then** 음원만 추출된 mp3 파일이 생성된다.
3. **Given** 사용자가 mp4를 선택하고 다운로드했을 때, **When** 저장 파일을 확인하면, **Then** 영상과 음성이 포함된 mp4 파일이 생성된다.

---

### User Story 3 - 다운로드 진행상황 실시간 표시 (Priority: P3)

다운로드가 진행되는 동안 사용자에게 진행률(퍼센트 또는 진행 바)과 현재 상태를 실시간으로 표시한다.

**Why this priority**: 진행상황 표시는 사용자 경험을 개선하지만, 없어도 핵심 기능은 동작한다. 파일이 다운로드되기만 해도 가치가 있다.

**Independent Test**: 다운로드 시작 후 UI에 진행률이 0%에서 100%로 변화하는 것을 확인.

**Acceptance Scenarios**:

1. **Given** 다운로드가 시작되었을 때, **When** UI를 확인하면, **Then** 진행률(%)과 현재 상태 메시지가 표시된다.
2. **Given** 다운로드가 진행 중일 때, **When** 진행률이 업데이트되면, **Then** UI의 진행 바가 실시간으로 갱신된다.
3. **Given** 다운로드가 완료되었을 때, **When** UI를 확인하면, **Then** "완료" 상태와 100% 진행률이 표시된다.

---

### Edge Cases

- 유효하지 않은 유튜브 URL(형식 오류, 존재하지 않는 영상, 비공개 영상, 지역 제한 영상)을 입력하면 어떻게 처리하는가?
- 다운로드 중 네트워크 연결이 끊어지면 어떻게 처리하는가?
- 저장 디스크 공간이 부족할 경우 어떻게 처리하는가?
- 동일한 파일이 이미 존재할 경우 덮어쓰기 또는 이름 변경을 어떻게 처리하는가?

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 유튜브 URL 입력 필드를 제공해야 한다 (MUST).
- **FR-002**: 시스템은 입력된 URL의 유효성을 검사하고 유튜브 URL 형식이 아닐 경우 오류 메시지를 표시해야 한다 (MUST).
- **FR-003**: 시스템은 유효한 URL 입력 시 영상 정보(제목, 길이, 썸네일 이미지)를 조회하여 표시해야 한다 (MUST).
- **FR-004**: 사용자는 다운로드 형식으로 mp4(동영상) 또는 mp3(음원) 중 하나를 선택할 수 있어야 한다 (MUST).
- **FR-005**: 시스템은 사용자가 선택한 형식으로 영상을 다운로드할 수 있어야 한다 (MUST).
- **FR-006**: 시스템은 다운로드 진행률(0~100%)을 실시간으로 표시해야 한다 (MUST).
- **FR-007**: 시스템은 다운로드 완료 후 사용자에게 완료 상태를 알려야 한다 (MUST).
- **FR-008**: 시스템은 다운로드 중 오류 발생 시 사용자에게 명확한 오류 메시지를 표시해야 한다 (MUST).
- **FR-009**: 다운로드된 파일은 서버 내 지정된 저장 경로에 저장되어야 한다 (MUST).

### Key Entities

- **다운로드 작업 (DownloadJob)**: 단일 다운로드 요청을 나타낸다. 속성: 원본 URL, 선택 형식, 상태(대기/진행중/완료/실패), 진행률(%), 저장 파일명.
- **영상 정보 (VideoInfo)**: 유튜브에서 조회한 영상 메타데이터. 속성: 제목, 썸네일 URL, 재생 시간, 사용 가능한 형식 목록.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 URL 입력부터 다운로드 완료까지 전체 과정을 5분 이내에 완료할 수 있다(영상 길이 제외, 조작 시간 기준).
- **SC-002**: 잘못된 URL 입력 시 3초 이내에 명확한 오류 메시지가 표시된다.
- **SC-003**: 다운로드 진행률이 1초 이하의 갱신 주기로 실시간 반영된다.
- **SC-004**: 선택한 형식(mp4/mp3)과 일치하는 파일이 100% 정확하게 저장된다.
- **SC-005**: 영상 정보(제목, 길이, 썸네일) 조회가 URL 입력 후 5초 이내에 완료된다.

## Assumptions

- 사용자는 안정적인 인터넷 연결 환경을 갖추고 있다.
- 다운로드된 파일은 서버 로컬 스토리지에 저장되며, 사용자는 별도 수단(파일 시스템 직접 접근)으로 파일을 가져간다(파일 전송 기능은 v1 범위 외).
- 동시 다운로드는 1건으로 제한한다(v1 범위).
- 지원 대상 URL은 유튜브 단일 영상이며, 재생목록·쇼츠·라이브 스트림은 v1 범위 외.
- mp4 저장 시 기본 화질은 최고 화질을 사용한다(화질 선택 기능은 v1 범위 외).
- 기술 스택: uv 기반 Python 패키지 관리, FastAPI(백엔드), Vue 3 + TypeScript + Vite(프론트엔드).
